# Security Policy

## Supported Versions

Only the latest release of OmniAgent Studio is actively supported with security updates and bug fixes.

| Version | Supported |
| ------- | --------- |
| >= 1.0.x| Yes       |
| < 1.0.0 | No        |

## Reporting a Vulnerability

We take the security of OmniAgent Studio seriously. If you find a security vulnerability, please do **not** open a public issue on GitHub. Doing so exposes the vulnerability publicly before a fix can be prepared.

Instead, please report security vulnerabilities by emailing us privately at:
📧 **demmagence@gmail.com**

Please include the following details in your email:
- **Description**: A detailed description of the vulnerability and its potential impact.
- **Steps to Reproduce**: Detailed step-by-step instructions (with code snippets, payloads, or screenshots if applicable) to reproduce the vulnerability.
- **Affected Environment**: OS, Node version, browser, or specific dependencies involved.

We will acknowledge receipt of your vulnerability report within **48 hours** and work on resolving it as quickly as possible.

## Disclosure Process

When a vulnerability is reported, we follow a responsible disclosure process:
1. We investigate the report to confirm the vulnerability and assess its severity.
2. We develop a patch or fix locally in private branches.
3. We coordinate a release containing the fix.
4. We publish a security advisory and credit the reporter (unless they request anonymity).

Thank you for helping keep OmniAgent Studio secure!
